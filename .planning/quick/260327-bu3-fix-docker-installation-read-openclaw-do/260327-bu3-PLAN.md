## Fix Docker Installation to Use Pre-Built Image and Correct OpenClaw Setup

**Purpose:** The current Docker install tries `docker compose up --build -d` which builds from source (slow, OOM-prone) and skips onboarding/config. This fix replaces it with the official flow: pull pre-built image, write complete .env, set gateway config via CLI, start gateway.

**Output:** Updated `docker_install.rs` with correct installation flow matching OpenClaw docs.

## Context

@src-tauri/src/install/docker_install.rs
@src-tauri/src/install/verify.rs
@src-tauri/src/commands/update.rs

The `update.rs` module already uses `ghcr.io/openclaw/openclaw` image constant and bollard-based image pull — this is the pattern to follow for consistency.

Current `docker_install.rs` problems:
1. `.env` is missing `OPENCLAW_IMAGE` and `OPENCLAW_GATEWAY_BIND` vars
2. `docker compose up --build -d` builds from Dockerfile (10+ min, often OOM)
3. No onboarding or gateway config is run — gateway starts with no config
4. Required subdirectories (identity/, agents/) are not created

**D) Replace Step 5** — instead of `docker compose up --build -d`, add new steps between .env write and gateway start:

1. **Pull pre-built image** using `docker pull ghcr.io/openclaw/openclaw:latest`:
   - Use `tokio::process::Command::new("docker").args(["pull", OPENCLAW_IMAGE])`
   - Stream stderr to log viewer (docker pull outputs progress to stderr)
   - Fail with clear error if pull fails

2. **Copy docker-compose.yml from repo to config dir** so update.rs can find it:
   - `tokio::fs::copy(repo_dir.join("docker-compose.yml"), config_dir.join("docker-compose.yml"))`

3. **Set gateway config via docker compose run** (3 commands, each `docker compose run --rm --no-deps -T --entrypoint node openclaw-gateway dist/index.js config set ...`):
   - `gateway.mode local`
   - `gateway.bind lan`
   - `gateway.controlUi.allowedOrigins '["http://localhost:18789","http://127.0.0.1:18789"]' --strict-json`
   - Use `-T` flag (no TTY) since these are non-interactive config commands
   - Run from `repo_dir` with `current_dir(&repo_dir)`
   - Each command's stdout/stderr emitted to log viewer
   - Non-fatal if config commands fail (emit warning, continue)

4. **Start gateway**: `docker compose up -d openclaw-gateway`
   - Same pattern as current code but WITHOUT `--build` flag
   - Use `current_dir(&repo_dir)`
   - Stream output to log viewer
