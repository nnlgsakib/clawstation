# Quick Task 260326-w1f: Add install dir selection, clean install, and cancel/pause

## Problem

1. Installation directory was hardcoded to `~/.openclaw` — users couldn't choose where to install
2. No way to clean the install directory before a fresh install
3. No way to cancel or pause an in-progress installation

## Tasks

### Task 1: Add install_dir to Rust backend

**Files:**
- `src-tauri/src/commands/install.rs` — added `install_dir: Option<String>` to `InstallRequest`, pass to `docker_install`
- `src-tauri/src/install/docker_install.rs` — accept `install_dir: Option<&str>`, use it instead of hardcoded `~/.openclaw`

### Task 2: Add clean_install_dir and cancel_install Tauri commands

**Files:**
- `src-tauri/src/commands/install.rs` — added `clean_install_dir(path)` and `cancel_install(install_dir)` commands
- `src-tauri/src/lib.rs` — registered both commands

`clean_install_dir`: removes the directory tree at the given path.
`cancel_install`: runs `docker compose down` in the repo dir to stop services.

### Task 3: Update frontend

**Files:**
- `src/hooks/use-install.ts` — added `installDir` to `InstallRequest`, exported `cleanInstallDir()` and `cancelInstall()` functions
- `src/stores/use-onboarding-store.ts` — added `installDir: string | null` and `setInstallDir`
- `src/components/install/step-install.tsx` — added folder picker (via `@tauri-apps/plugin-dialog`), clean install button, cancel button

## must_haves

- truths: TypeScript and Rust compile cleanly
- truths: Users can pick a custom install directory
- truths: Users can clean the install directory before installing
- truths: Users can cancel an in-progress installation
