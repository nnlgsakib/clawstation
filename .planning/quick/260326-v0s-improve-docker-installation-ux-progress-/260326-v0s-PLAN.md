# Quick Task 260326-v0s: Improve Docker installation UX

## Problem

Docker installation UX was poor:
1. **Log viewer noise** — hundreds of "Downloading" lines with hex layer IDs like `[aab3b37e4807]`
2. **Wrong layer percentages** — `percentage` was overall pull progress mapped to 20-50%, not per-layer download progress
3. **No friendly descriptions** — raw Docker status strings like "Downloading", "Verifying Checksum"
4. **No progress bar** — no visual indication of installation step progression

## Tasks

### Task 1: Fix Rust backend event emission

**Files:** `src-tauri/src/install/docker_install.rs`

**Action:**
- Added `layerPercentage` field to `DockerLayerProgressEvent` — actual per-layer download % (0-100)
- Only emit `docker-log-output` for meaningful events: "Verifying Checksum", "Download complete", "Extracting", "Pull complete", "Already exists" — not every "Downloading" tick
- Shortened layer IDs in log output to 8 chars

### Task 2: Update frontend hook and components

**Files:**
- `src/hooks/use-docker-layer-progress.ts` — accept `layerPercentage`, add `friendlyStatus()` mapping
- `src/components/ui/layer-progress.tsx` — show status icons, layer count, use per-layer percentage
- `src/hooks/use-docker-logs.ts` — add `lineCount`, deduplicate consecutive identical lines
- `src/components/ui/log-viewer.tsx` — add header with event count
- `src/components/install/step-install.tsx` — add `InstallProgressBar` with step indicators, improved layout

## must_haves

- truths: TypeScript and Rust compile cleanly
- truths: Docker installation log shows only meaningful events (not noise)
- truths: Layer progress shows actual per-layer download percentages
- artifacts: docker_install.rs, use-docker-layer-progress.ts, layer-progress.tsx, use-docker-logs.ts, log-viewer.tsx, step-install.tsx
