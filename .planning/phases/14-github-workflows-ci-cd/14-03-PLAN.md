## Current Release Workflow Analysis

The existing `.github/workflows/release.yml` has these issues preventing proper artifact release:

### Issue 1: Artifact download flattening may lose structure
Current code:
```yaml
- name: Download all artifacts
  uses: actions/download-artifact@v4
  with:
    path: artifacts
    merge-multiple: true
```
With `merge-multiple: true`, all artifact files are flattened into `artifacts/`. However, if any artifact contains subdirectories (common with Tauri bundle output), those subdirectories are preserved, causing `artifacts/*.sha256` glob to miss files in nested dirs.

### Issue 2: Release files pattern is not recursive
Current code:
```yaml
files: artifacts/*
```
This only matches files directly in `artifacts/`, not files in subdirectories. Tauri bundles often have nested structure.

### Issue 3: Build job artifact collection may miss some bundle types
Current code copies from:
```bash
BUNDLE_DIR="src-tauri/target/${{ matrix.rust_target }}/release/bundle"
```
But Tauri v2 bundle output structure varies by platform:
- Linux: `bundle/appimage/*.AppImage`, `bundle/deb/*.deb`
- Windows: `bundle/msi/*.msi`, `bundle/nsis/*.exe`
- macOS: `bundle/dmg/*.dmg`, `bundle/macos/*.app`

The `find` command with `-type f` and name patterns should catch these, but the path construction may be wrong if the target directory structure differs.

### Issue 4: Release body checksum generation path
Current code:
```yaml
for f in artifacts/*.sha256; do
```
This won't find `.sha256` files if they're in subdirectories after download.

## Research: GitHub Actions download-artifact v4 behavior

With `merge-multiple: true`:
- All artifacts are downloaded to the specified path
- Files from each artifact are merged into the same directory
- If two artifacts have files with the same name, the last one wins
- Subdirectories within artifacts are preserved

The safest approach: Download without merge-multiple, then use a recursive glob or explicit path handling.

### 1. Build job: Fix artifact upload structure
Keep the current build matrix (5 platforms). Fix the "Prepare release assets" step to ensure files are collected correctly:

```yaml
- name: Prepare release assets
  shell: bash
  run: |
    BUNDLE_DIR="src-tauri/target/${{ matrix.rust_target }}/release/bundle"
    mkdir -p release-artifacts
    echo "Looking for bundles in: $BUNDLE_DIR"
    find "$BUNDLE_DIR" -type f \( \
      -name "*.AppImage" -o -name "*.deb" -o -name "*.rpm" \
      -o -name "*.msi" -o -name "*.exe" -o -name "*.dmg" \
      -o -name "*.zip" -o -name "*.sig" \
    \) -exec cp {} release-artifacts/ \;
    # Generate checksums
    cd release-artifacts
    for f in *.AppImage *.deb *.rpm *.msi *.exe *.dmg *.zip 2>/dev/null; do
      [ -f "$f" ] || continue
      if command -v sha256sum &> /dev/null; then
        sha256sum "$f" > "$f.sha256"
      else
        shasum -a 256 "$f" > "$f.sha256"
      fi
    done
    ls -la
```

Upload with flat structure:

- name: Upload artifacts

### 2. Release job: Fix artifact download and release creation
Download artifacts WITHOUT merge-multiple to preserve structure, then collect all files:

```yaml
- name: Download all artifacts
  uses: actions/download-artifact@v4
  with:
    path: downloaded-artifacts

- name: Collect release files
  run: |
    mkdir -p release-files
    # Flatten all downloaded artifacts into one directory
    find downloaded-artifacts -type f \( \
      -name "*.AppImage" -o -name "*.deb" -o -name "*.rpm" \
      -o -name "*.msi" -o -name "*.exe" -o -name "*.dmg" \
      -o -name "*.zip" -o -name "*.sig" -o -name "*.sha256" \
    \) -exec cp {} release-files/ \;
    echo "--- Release files ---"
    ls -la release-files/
```

### 3. Release body: Fix checksum table generation
Generate release body from the collected release-files directory:

```yaml
- name: Create release body
  run: |
    {
      echo "## ClawStation ${{ github.ref_name }}"
      echo ""
      echo "### Downloads"
      echo ""
      echo "| File | SHA-256 |"
      echo "|------|---------|"
      for f in release-files/*.sha256; do
        [ -f "$f" ] || continue
        fname=$(basename "$f" .sha256)
        hash=$(awk '{print $1}' "$f")
        echo "| \`$fname\` | \`${hash:0:16}...\` |"
      done
      echo ""
      echo "<details><summary>Full checksums</summary>"
      echo ""
      echo '```'
      cat release-files/*.sha256 2>/dev/null || echo "No checksums"
      echo '```'
      echo ""
      echo "</details>"
    } > release-body.md
    cat release-body.md
```

### 4. GitHub Release: Fix files pattern
Use the collected release-files directory:

```
