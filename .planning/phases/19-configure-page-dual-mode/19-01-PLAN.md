  <action>
    Create `src/components/config/json-editor.tsx` with:

    1. A `JsonEditor` component that:
       - Reads `storeConfig` from `useConfigStore`
       - Renders a `<textarea>` pre-filled with `JSON.stringify(storeConfig, null, 2)`
       - Tracks local JSON text in `useState` — initialized from store config
       - On text change: updates local state, attempts `JSON.parse()` — if parse fails, shows inline error below textarea (red text, "Invalid JSON: {error message}")
       - Has a "Format" button that pretty-prints the JSON (re-stringify with 2-space indent)
       - Shows a `parseError` state string when JSON is invalid

    2. Props interface:
       ```typescript
       interface JsonEditorProps {
         onSave: (config: Record<string, unknown>) => void;
         isSaving: boolean;
       }
       ```

    3. The component does NOT handle save directly — it calls `onSave(parsedConfig)` when the parent triggers save. The parent (configure.tsx) handles the actual save logic.

    4. Styling:
       - Use `font-mono` on the textarea
       - Use existing Tailwind classes matching the app style: `border-input bg-background rounded-md`
       - Min height: `min-h-[400px]` for comfortable editing
       - Error display: `<p className="text-xs text-destructive mt-1">` below the textarea
       - Wrap in a `<Card>` with `<CardHeader>` titled "JSON Editor"

    5. Do NOT use any external code editor library (CodeMirror, Monaco) — use a plain styled `<textarea>` with `font-mono`. This keeps the bundle small and avoids native module issues with Tauri.

    6. Export both the component and the props type.
  </action>
