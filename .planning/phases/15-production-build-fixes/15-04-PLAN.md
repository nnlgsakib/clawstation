<objective>
Fix production build PATH/environment issue where spawned processes can't find npm, pnpm, yarn, or globally-installed binaries like `openclaw`.

Purpose: In dev mode, the app inherits the terminal's full PATH (which includes Node.js global bin directories like `C:\Users\User\AppData\Roaming\npm`). In production builds launched from the desktop, the app's environment doesn't include these directories. This causes:
- `pnpm` global bin not in PATH → install fails
- `yarn` installs openclaw but can't find it → verification fails
- `openclaw --version` returns "command not found" after successful npm install
- `pnpm exec openclaw` returns ERR_PNPM_RECURSIVE_EXEC_NO_PACKAGE

Output: `silent_cmd()` augmented with Node.js PATH on Windows. All spawned processes inherit the user's package manager directories.
</objective>

**Root cause:** `silent_cmd()` creates `tokio::process::Command::new(program)` which inherits the parent process's environment. But on Windows production builds, the parent environment lacks Node.js global bin directories. When `cmd /c openclaw --version` runs, cmd.exe can't find `openclaw` because it's installed to `%APPDATA%\npm` which isn't in PATH.

**The fix pattern:**
```rust
// Add to silent_cmd() on Windows:
// 1. Get current PATH
// 2. Detect Node.js global bin dirs (%APPDATA%\npm, %LOCALAPPDATA%\pnpm, .yarn\bin)
// 3. Prepend them to PATH via .env("PATH", augmented_path)
```

    Then modify `silent_cmd()` to apply these environment variables:
    ```rust
    pub fn silent_cmd(program: &str) -> tokio::process::Command {
        let mut cmd = tokio::process::Command::new(program);
        cmd.stdout(Stdio::piped()).stderr(Stdio::piped());
        #[cfg(target_os = "windows")]
        {
            cmd.creation_flags(0x08000000); // CREATE_NO_WINDOW
            // Augment PATH with Node.js global bin directories
            for (key, value) in nodejs_path_env() {
                cmd.env(key, value);
            }
        }
        cmd
    }
    ```

    ```rust
    async fn find_openclaw_binary() -> Option<String> {
        // Common paths where npm/yarn/pnpm install global binaries on Windows
        let candidates: Vec<String> = if cfg!(target_os = "windows") {
            let appdata = std::env::var("APPDATA").unwrap_or_default();
            let localappdata = std::env::var("LOCALAPPDATA").unwrap_or_default();
            vec![
                format!("{appdata}\\npm\\openclaw.cmd"),
                format!("{appdata}\\npm\\openclaw"),
                format!("{localappdata}\\pnpm\\openclaw.cmd"),
                format!("{localappdata}\\pnpm\\openclaw"),
                "C:\\Program Files\\nodejs\\openclaw.cmd".to_string(),
                "C:\\Program Files\\nodejs\\openclaw".to_string(),
            ]
        } else {
            let home = std::env::var("HOME").unwrap_or_default();
            vec![
                format!("{home}/.npm/bin/openclaw"),
                format!("{home}/.local/share/pnpm/openclaw"),
                "/usr/local/bin/openclaw".to_string(),
            ]
        };

        for path in &candidates {
            if tokio::fs::metadata(path).await.is_ok() {
                return Some(path.clone());
            }
        }
        None
    }
    ```

    ```rust
        // Try direct command first
        let mut cmd = silent_cmd("openclaw");
        cmd.arg("--version");
        if let Ok(output) = run_with_timeout(&mut cmd, QUICK_TIMEOUT).await {
            if output.status.success() {
                let version_str = String::from_utf8_lossy(&output.stdout).trim().to_string();
    ```
