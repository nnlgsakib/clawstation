## v1 Requirements (All Complete)

### Installation

- [x] **INST-01**: User can install OpenClaw via one-click Docker setup with system requirements pre-check — Phase 3

- [x] **INST-04**: App verifies installation succeeded by running a health check post-install — Phase 3
- [x] **INST-05**: App shows first-run onboarding (3-step: system check → install → ready) — Phase 3

### Configuration

- [x] **CONF-01**: User can configure OpenClaw provider/model selection visually — Phase 4
- [x] **CONF-02**: User can configure sandbox settings visually (mode, scope, workspace access) — Phase 4
- [x] **CONF-03**: User can configure tool policies visually (which tools are allowed/denied) — Phase 4
- [x] **CONF-04**: User can configure agent defaults visually (sandbox mode, autonomy) — Phase 4
- [x] **CONF-05**: Config editor validates changes before writing (schema validation) — Phase 4
- [x] **CONF-06**: User can configure bind mounts for sandbox (directory picker) — Phase 4

### Sandboxing

- [x] **SAND-02**: User can choose sandbox backend (Docker, SSH, OpenShell) — Phase 4
- [x] **SAND-03**: User can set sandbox mode (off, non-main, all) — Phase 4
- [x] **SAND-04**: User can set workspace access level (none, read-only, read-write) — Phase 4
- [x] **SAND-05**: User can configure sandbox network policy (none, custom) — Phase 4
- [x] **SAND-06**: App runs sandbox setup scripts automatically when sandbox is enabled — Phase 4

- [x] **LIFE-01**: User can update OpenClaw to newer versions with one click — Phase 6

### Error Handling
