<task type="auto" tdd="true">
  <name>Task 1: Add tokio-tungstenite dependency and create WebSocket client</name>
  <files>src-tauri/Cargo.toml, src-tauri/src/commands/gateway_ws.rs, src-tauri/src/lib.rs</files>
  <read_first>src-tauri/Cargo.toml, src-tauri/src/commands/config.rs</read_first>
  <behavior>
    - gateway_ws_connect: establishes WebSocket connection to ws://127.0.0.1:{port}, stores connection handle in AppState, emits gateway-ws-status events (connected/disconnected/error), spawns message reader task
    - gateway_ws_call: sends JSON-RPC style request to Gateway, waits for response with matching id, returns result. Supports: config.get, config.apply, config.patch, sessions.list, sessions.send, node.list
    - gateway_ws_disconnect: closes WebSocket connection, clears state
    - Auto-reconnect: if connection drops, emits disconnected event, tries reconnect after 2s, 4s, 8s (exponential backoff, max 30s)
    - Message format: { "id": "uuid", "method": "config.get", "params": {} } → { "id": "uuid", "result": {...} }
  </behavior>
  <action>
    In Cargo.toml, add:
    ```toml
    tokio-tungstenite = "0.26"
    uuid = { version = "1", features = ["v4"] }
    ```
    
    Create src-tauri/src/commands/gateway_ws.rs:
    
    Define GatewayWsState in state.rs (add to AppState):
    ```rust
    pub struct AppState {
        // existing fields...
        pub gateway_ws: Option<GatewayWsHandle>,
    }
    
    pub struct GatewayWsHandle {
        pub write: futures::stream::SplitSink<WebSocketStream<MaybeTlsStream<TcpStream>>, tungstenite::Message>,
        pub pending_requests: Arc<Mutex<HashMap<String, tokio::sync::oneshot::Sender<serde_json::Value>>>>,
    }
    ```
    
    Implement gateway_ws_connect:
    - Accept app: AppHandle, port: Option<u16> (default 18789)
    - Connect to ws://127.0.0.1:{port} using tokio_tungstenite::connect_async
    - Split stream into read/write halves
    - Store write handle in AppState
    - Spawn reader task that:
      - Reads messages from read half
      - Parses JSON, looks up pending_requests by id
      - Sends response via oneshot channel
      - On connection close, emits "gateway-ws-status" with disconnected=true
    - Emit "gateway-ws-status" with connected=true
    - Return { connected: true }
    
    Implement gateway_ws_call:
    - Accept app: AppHandle, method: String, params: serde_json::Value
    - Generate UUID for request id
    - Create oneshot channel for response
    - Store sender in pending_requests
    - Send JSON message via write handle: { id, method, params }
    - Wait for response with 10s timeout
    - Return response result
    
    Implement gateway_ws_disconnect:
    - Take write handle from AppState
    - Send close frame
    - Clear AppState.gateway_ws
    
    In lib.rs, add to invoke_handler:
    - commands::gateway_ws::gateway_ws_connect
    - commands::gateway_ws::gateway_ws_call
    - commands::gateway_ws::gateway_ws_disconnect
  </action>
  <verify>cargo check passes, WebSocket connection can be established to running Gateway</verify>
  <done>WebSocket client connects to Gateway and can make API calls</done>

<task type="auto" tdd="true">

  <files>src/hooks/use-gateway.ts, src/stores/use-gateway-store.ts</files>

  <behavior>
    - useGatewayConnection: connects to Gateway on mount, auto-reconnects, returns { connected, connecting, error }
    - useGatewayCall: wraps gateway_ws_call with loading/error states, accepts method and params, returns UseQueryResult
    - useGatewayStore: Zustand store tracking connection status, last error, reconnect attempts
    - useGatewayConfig: convenience hook that calls config.get via Gateway WebSocket
    - useGatewaySessions: convenience hook that calls sessions.list via Gateway WebSocket
    - All hooks use TanStack Query for caching and background refetch
  </behavior>
  <action>
    Create src/stores/use-gateway-store.ts:
    ```typescript
    interface GatewayState {
      connected: boolean;
      connecting: boolean;
      error: string | null;
      reconnectAttempts: number;
      setConnected: () => void;
      setConnecting: () => void;
      setError: (error: string) => void;
      reset: () => void;
    }
    export const useGatewayStore = create<GatewayState>(...)
    ```
    
    Create src/hooks/use-gateway.ts:
    
    useGatewayConnection():
    - On mount, invoke gateway_ws_connect
    - Listen to gateway-ws-status events via Tauri event API
    - Update useGatewayStore on status changes
    - On disconnect, attempt reconnect with exponential backoff (2s, 4s, 8s, max 30s)
    - Cleanup: invoke gateway_ws_disconnect on unmount
    - Returns: { connected: boolean, connecting: boolean, error: string | null }
    
    useGatewayCall<T>(method: string, params?: object):
    - Uses useQuery from TanStack Query
    - Query key: ["gateway", method, params]
    - Query fn: invoke("gateway_ws_call", { method, params })
    - Enabled only when gateway is connected
    - staleTime: 30s for config, 5s for sessions
    - Returns: UseQueryResult<T>
    
    useGatewayConfig():
    - Calls useGatewayCall("config.get", {})
    - Returns config with hash for optimistic locking
    
    useGatewaySessions():
    - Calls useGatewayCall("sessions.list", {})
    - Returns agent sessions array
  </action>
