Channel config structure (from OpenClaw docs — per-channel fields):
```typescript
interface ChannelConfig {
  enabled?: boolean;
  botToken?: string;      // Telegram, Slack
  appToken?: string;      // Slack (required alongside botToken)
  token?: string;         // Discord (uses "token", not "botToken")
  allowFrom?: string[];   // Who can message (phone numbers, user IDs)
  dmPolicy?: "pairing" | "allowlist" | "open" | "disabled";
  groups?: Record<string, { requireMention?: boolean }>; // Telegram groups
  healthMonitor?: { enabled?: boolean }; // Channel health monitoring
}

// Per-provider fields (from official docs):
// WhatsApp: allowFrom, dmPolicy, groups
// Telegram: enabled, botToken, allowFrom, dmPolicy, groups, healthMonitor
// Discord: token, allowFrom, dmPolicy, guilds
// Slack: botToken, appToken, allowFrom, dmPolicy
// Signal: (requires signal-cli)
// Microsoft Teams: (plugin, installed separately)
```
