    - check_nodejs: runs `node --version`, parses semver, returns { installed: bool, version: String, meetsMinimum: bool, meetsRecommended: bool } where minimum is 22.14.0, recommended is 24.0.0
    - install_openclaw: runs platform-specific install script (per D-02):
      - Windows: `iwr -useb https://openclaw.ai/install.ps1 | iex` (PowerShell)
      - macOS/Linux: `curl -fsSL https://openclaw.ai/install.sh | bash`
    - Emits install-output events with stdout lines for progress
    - After install, runs `openclaw --version` to confirm
    - Returns installed version string on success
    - On failure, returns descriptive error string (e.g., "Node.js not found in PATH", "Install script failed: EACCES")
  </behavior>
  <action>
    Create src-tauri/src/commands/nodejs.rs following the pattern from docker.rs.
    
    Define struct NodeJsInfo with serde derives:
    ```rust
    #[derive(Debug, Clone, Serialize, Deserialize)]
    #[serde(rename_all = "camelCase")]
    pub struct NodeJsInfo {
        pub installed: bool,
        pub version: Option<String>,
        pub meets_minimum: bool,     // >= 22.14.0
        pub meets_recommended: bool, // >= 24.0.0
    }
    ```
    
    Implement check_nodejs:
    - Use tokio::process::Command to run "node --version"
    - Parse output like "v24.0.0" or "v22.14.0" into semver
    - Return NodeJsInfo with installed=true, version, meets_minimum=(major>22 || (major==22 && minor>=14)), meets_recommended=(major>=24)
    - On error (command not found), return NodeJsInfo { installed: false, version: None, meets_minimum: false, meets_recommended: false }
    
    Implement install_openclaw:
    - Accept app: tauri::AppHandle parameter
    - Detect platform via std::env::consts::OS
    - Windows: spawn PowerShell with `iwr -useb https://openclaw.ai/install.ps1 | iex`
    - Unix: spawn shell with `curl -fsSL https://openclaw.ai/install.sh | bash`
    - Stream stdout lines via app.emit("install-output", line)
    - On exit, run "openclaw --version" to confirm installation
    - Return installed version string
    
    In src-tauri/src/lib.rs, add to invoke_handler:
    - commands::nodejs::check_nodejs
    - commands::nodejs::install_openclaw

<task type="auto" tdd="true">
  <name>Task 2: Create Gateway process management commands</name>
  <files>src-tauri/src/commands/gateway.rs, src-tauri/src/lib.rs</files>
  <read_first>src-tauri/src/commands/install.rs, src-tauri/src/state.rs</read_first>
  <behavior>
    - start_gateway: spawns "openclaw gateway --verbose" as child process, stores Child handle in AppState, emits gateway-output events with stdout/stderr, returns { started: true, port: 18789 }
    - stop_gateway: kills the stored child process, clears state, emits gateway-stopped event, returns { stopped: true }
    - get_gateway_status: checks if child process is still alive (try_wait), checks if port 18789 is listening (TCP connect), returns { running: bool, port: u16, pid: Option<u32> }
    - Process managed with process-wrap for cross-platform kill-on-drop behavior
    - Gateway output streamed in real-time via app.emit("gateway-output", { line: String, stream: "stdout"|"stderr" })
  </behavior>
  <action>
    Create src-tauri/src/commands/gateway.rs following the pattern from install.rs.
    
    Add GatewayProcess to AppState (in state.rs):
    ```rust
    pub struct AppState {
        // existing fields...
        pub gateway_process: Option<tokio::sync::Mutex<tokio::process::Child>>,
    }
    ```
    
    Implement start_gateway:
    - Accept app: tauri::AppHandle, port: Option<u16>
    - Build command: "openclaw gateway --verbose --port {port}" (default 18789)
    - Use tokio::process::Command with stdout/stderr piped
    - Spawn and store Child in AppState.gateway_process
    - Spawn async task to read stdout line by line and emit "gateway-output" events
    - Spawn async task to read stderr and emit "gateway-output" events with stream="stderr"
    - Return GatewayStatus { started: true, port }
    
    Implement stop_gateway:
    - Take app: tauri::AppHandle
    - Get AppState, take gateway_process
    - Call child.kill().await (or process.kill() for cross-platform)
    - Emit "gateway-stopped" event
    - Return { stopped: true }
    
    Implement get_gateway_status:
    - Check if gateway_process exists in AppState
    - Try TCP connect to 127.0.0.1:18789 to verify Gateway is actually listening
    - Return GatewayStatus { running, port, pid }
    
    Define struct GatewayStatus:
    ```rust
    #[derive(Debug, Clone, Serialize, Deserialize)]
    #[serde(rename_all = "camelCase")]
    pub struct GatewayStatus {
        pub running: bool,
        pub port: u16,
        pub pid: Option<u32>,
    }
    ```
    
    In src-tauri/src/lib.rs, add to invoke_handler:
    - commands::gateway::start_gateway
    - commands::gateway::stop_gateway
    - commands::gateway::get_gateway_status
  </action>
